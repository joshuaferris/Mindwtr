# Mindwtr 0.8.4

## Highlights

- Clarified the shipped mobile calendar sync behavior in the README and roadmap language.
- Added ADR 0009 so the SQLite-to-JSON sync bridge is explicit in the architecture and wiki docs.
- Hardened sync regression coverage so device-local sync history does not trigger unnecessary remote writes.

## Docs & Sync Maintenance

- Documented that SQLite remains the primary local store while `data.json` stays the sync and backup snapshot.
- Clarified that sync keeps device-local diagnostics out of remote payloads and requeues instead of overwriting fresher local edits.
- Updated release and roadmap wording to reflect the shipped mobile calendar sync behavior.

## Reliability

- Added desktop and mobile runtime tests that lock in the sanitized-write behavior for device-local sync history.
- Added a code comment to keep external `setState` writes flowing through the store normalization path.

## Full Change List (since `v0.8.3`)

- docs(sync): codify sqlite-json bridge
- docs(readme): reflect shipped calendar sync
- test(sync): lock device-local history writes
